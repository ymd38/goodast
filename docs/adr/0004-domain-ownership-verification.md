# ADR-0004: スキャン前にドメイン所有確認を必須とする

Status: Accepted
Date: 2026-06-27

## Context

goodast はUIから簡単にスキャンを開始できる。利便性が高い反面、**他人が管理するサイトを無断でスキャンする悪用**が容易になるリスクがある。無断の脆弱性スキャンは法的・倫理的に問題となりうる。安全装置として、スキャン対象が「ユーザー自身が管理するサイト」であることを保証する必要がある。

## Decision

スキャン対象ドメインの**所有確認を通すまでスキャンを実行不可**とする。

- **ファイル設置（優先）**: `/.well-known/goodast-verify/<token>` にファイルを設置してもらい、HTTPアクセスで確認。サーバにアクセスできれば初心者でも対応可能。
- **DNS TXT（オプション）**: `goodast-verify=<token>` のTXTレコードをDNS設定（レジストラ/Cloudflare等）に追加してもらい、DNSクエリで確認。エンジニア向けの代替手段。
- **ローカル環境は確認スキップ**: `localhost`・`127.0.0.1`・`::1`・`*.local` は所有が自明なため確認不要。開発・テスト（OWASP Juice Shop等）の障壁を排除する。

## Consequences

- **利点**: 不正スキャンを防ぐ明確な安全装置。OSSとして公開する際の法的・倫理的な防御線になる。初心者にはファイル設置、エンジニアにはDNS TXTと、層に応じた手段を提供できる。
- **トレードオフ**: 正規ユーザーにも一手間が発生する。→ ローカル除外と、わかりやすい設置ガイド（UIで図解）で摩擦を最小化する。
- スコープallowlist・危険パス除外・レート制限と合わせて多層のガードレールを構成する。
