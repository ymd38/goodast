# ADR-0003: 認証情報はアプリケーションレイヤーで暗号化する

Status: Accepted
Date: 2026-06-27

## Context

goodast は認証後ページのスキャンのため、ユーザーのログイン済みセッション（Cookie / Authorizationヘッダ）を保管する。これは機微情報であり、暗号化保存が必須。実装位置の選択肢は2つ。

1. **PostgreSQL pgcrypto**: DB拡張機能でDB側で暗号化。
2. **アプリケーションレイヤー**: Goコード側で暗号化してからDBに保存。

## Decision

**アプリケーションレイヤー**で暗号化する。pgcryptoは使わない。

- 認証情報は `api/` 側で暗号化してから `scan_credentials.enc_headers`（bytea）に保存。
- 平文ログ禁止。APIレスポンスでは生値を返さずマスク表示。

## Consequences

- **利点**: DB固有機能への依存が減り、テスト・移植・DB差し替えが容易。暗号化ロジックがコード側にあり、鍵管理・ローテーションをアプリで一元管理できる。
- **トレードオフ**: DB単体クエリで復号できない（アプリ経由必須）。これはセキュリティ上はむしろ望ましい。
- 鍵の保管（環境変数・シークレットマネージャ）は別途運用設計が必要。
- 同方式はフェーズ2のBYOK APIキー暗号化にもそのまま適用する。
